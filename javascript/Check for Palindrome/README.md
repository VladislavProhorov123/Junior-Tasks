# Check for Palindrome

## Суть задачи

Нужно определить, является ли строка или число палиндромом.

Палиндром — это значение, которое одинаково читается слева направо и справа налево.

Примеры:

```js
"121" => true
"1331" => true
"911" => false
"racecar" => true
```

---

## Условие

Дана строка.

Необходимо вернуть результат проверки: является ли она палиндромом.

Пример решения:

```js
function isPalindrome(str) {
  const reversed = str.split('').reverse().join('');

  return str === reversed;
}
```

---

## Разбор решения

### Шаг 1. Получаем исходную строку

```js
str = "121"
```

---

### Шаг 2. Разбиваем строку на массив символов

```js
str.split('')
```

Результат:

```js
['1', '2', '1']
```

---

### Шаг 3. Разворачиваем массив

```js
['1', '2', '1'].reverse()
```

Результат:

```js
['1', '2', '1']
```

---

### Шаг 4. Собираем обратно в строку

```js
['1', '2', '1'].join('')
```

Результат:

```js
"121"
```

---

### Шаг 5. Сравниваем строки

```js
str === reversed
```

То есть:

```js
"121" === "121"
```

Результат:

```js
true
```

---

## Визуализация

Для строки:

```js
"911"
```

После реверса:

```js
"119"
```

Сравнение:

```js
"911" === "119"
```

Результат:

```js
false
```

---

## Сложность решения

### Время

```js
O(n)
```

Где `n` — количество символов в строке.

Методы:

```js
split()
reverse()
join()
```

проходят по данным.

---

### Память

```js
O(n)
```

Создаётся новый массив и новая строка.

---

## Решение через Two Pointers

Это решение часто считается более сильным на собеседовании.

```js
function isPalindrome(str) {
  let left = 0;
  let right = str.length - 1;

  while (left < right) {
    if (str[left] !== str[right]) {
      return false;
    }

    left++;
    right--;
  }

  return true;
}
```

---

## Как работает Two Pointers

Строка:

```js
"1221"
```

Начальное состояние:

```text
left  = 0
right = 3
```

Сравниваем:

```text
1 === 1
```

Сдвигаем указатели:

```text
left++
right--
```

Получаем:

```text
left  = 1
right = 2
```

Сравниваем:

```text
2 === 2
```

Все проверки пройдены.

Результат:

```js
true
```

---

## Сложность Two Pointers

### Время

```js
O(n)
```

### Память

```js
O(1)
```

Дополнительные структуры данных не создаются.

## Частые вопросы на собеседовании

### Почему решение через Two Pointers лучше?

Потому что не создаётся новая строка и новый массив.

Память:

```js
O(1)
```

вместо

```js
O(n)
```

---

### Можно ли решить через reverse()?

Да.

```js
str === str.split('').reverse().join('')
```

Это самое популярное решение.

---

### Что будет с пустой строкой?

Она считается палиндромом.

```js
"" => true
```

---

### Что будет со строкой из одного символа?

Она тоже является палиндромом.

```js
"a" => true
```

---

### Что выбрал бы на собеседовании?

1. Сначала показал бы решение через `reverse()`.
2. После этого предложил бы оптимизацию через Two Pointers.

Такой подход показывает умение сначала решить задачу, а затем улучшить решение.
